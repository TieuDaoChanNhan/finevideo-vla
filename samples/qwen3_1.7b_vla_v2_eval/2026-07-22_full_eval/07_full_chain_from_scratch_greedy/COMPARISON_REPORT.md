# full_chain_from_scratch, greedy -- language vs. decoded media

## Language description fed to the model (the ONLY input, no real tokens given)

```
### Title: Home cooking tutorial
### Context: A person chops vegetables on a cutting board in a kitchen.
### Keywords: cooking, kitchen, food preparation
```

Everything below this point was generated by the model itself (greedy decoding, 2000 max new tokens) --
no ground-truth seed2/cosmos/snac/agent tokens were given at any point.

## Model's own text output (for comparison against the language above)

- `<caption>`: **"The person is cutting vegetables on a cutting board."** -- matches the input context closely.
- `<speech>`: **"1.5g of fresh vegetables."** -- topically on-theme (vegetables) but a slightly odd, recipe-ish phrase to be "spoken".

## Modality transition order (what the model actually walked through)

```
caption -> seed2 -> snac -> speech -> cosmos -> snac -> snac -> seed2 -> snac -> snac -> snac -> cosmos -> snac -> seed2 -> cosmos -> snac -> cosmos (cut off at 2000-token budget)
```

6 distinct modality switches -- this is the "modality transitions: FAIL" failure mode from the v1 model
(CLAUDE.md) not reproducing here; the model moves freely instead of getting stuck in one modality.

## Decoded media

### seed2 -> image (3 blocks found, all 3 IDENTICAL)

**Finding:** all 3 `<seed2>...</seed2>` blocks the model generated across this 2000-token run contain the
exact same 32 token ids, in the exact same order. This is a macro-level repetition -- the same failure
*category* as greedy's known token-level repetition inside long `cosmos` runs (REPORT.md #31), just at the
scale of "repeat a whole 32-token image block" instead of "repeat 1 token". Practical effect: only 1 image
worth of information is being shown 3 times, not 3 different frames/views.

- `decoded_seed2_0.png`, `decoded_seed2_1.png`, `decoded_seed2_2.png` -- byte-identical output (confirmed:
  same input tokens -> same fixed-latent diffusion decode).

### cosmos -> video (787 tokens generated, 3 full 200-token chunks decoded + 1 partial 187-token chunk dropped)

- `decoded_cosmos_chunk0.mp4`, `decoded_cosmos_chunk1.mp4`, `decoded_cosmos_chunk2.mp4`

### snac -> audio (75 tokens across multiple `<snac>...</snac>` blocks, all well-formed)

- `decoded_snac.wav`

### agent -> 3D pose: NOT REACHED

Zero `<fps_N>` tokens anywhere in the 2000-token generation. The model spent its budget cycling through
caption/seed2/snac/speech/cosmos (cosmos alone burned 787 of 2000 tokens -- almost 40%) and never opened an
`<agent>` block. Consistent with the "cosmos dominates generation" finding (REPORT.md #31 follow-up):
cosmos's fixed ~200-token/chunk cost can crowd out other modalities before they get a turn, and this run is
a direct example of the agent modality specifically losing that competition.

## Bottom line vs. the language description

The model clearly *understood the topic* (caption and speech both stayed on "vegetables"/"cutting board"/
kitchen context) and *executed correct modality-transition structure* (6 switches, no crash, well-formed
tags throughout). What it did NOT do: produce 3 visually distinct images for 3 separate seed2 openings
(all identical -- a real generation-diversity gap under greedy), or ever reach the agent/pose modality
within budget.
